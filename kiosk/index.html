Authentication Design

Two paths, one kiosk

The method is chosen by the employee — the kiosk supports both simultaneously.

🖐️

Fingerprint

Office · Managers · Front of house
1
Place finger on Suprema reader
2
On-device match — <1 second
3
Employee identified → home screen
🔒 Biometric stays on device. Base44 stores only a template ID — never the fingerprint itself.
📡 + 🔢

NFC Wristband + PIN

Kitchen · Cooks · Gloved staff
1
Tap silicone wristband on reader
2
Enter 4-digit PIN on numpad
3
Employee identified → home screen
Works with wet hands, gloves, burn bandages. NFC UID is not biometric — zero privacy liability.

Technical Architecture

How it's built

🖥️ Windows Mini-PC (local)
Electron App — runs your existing React/Vite UI fullscreen. No browser bar, no taskbar, boots straight to kiosk.
React 18 Tailwind Electron 33
🔌 Hardware Abstraction Layer
Electron main process talks to USB devices, sends events to UI over IPC. Fingerprint template match happens on-device only.
↕ HTTPS
☁️ Base44 Backend
7 new functions: verifyEmployee, clockIn, clockOut, verifyPin, enrollNfc, enrollFingerprint, getTodayPunches
🏗️ Mac Dev → Windows Prod
1
Develop on Mac — NFC works natively, fingerprint uses a dev mock (employee picker)
2
Push to GitHub → CI auto-builds Windows .exe
3
Run installer once per machine. Auto-updates on every push from that point.
🔒 Biometric Privacy
Fingerprint templates stored encrypted, locally on the mini-PC only.
Base44 stores a template ID string — never biometric data.
NFC UIDs are not biometric — no liability.
PINs stored as bcrypt hash server-side.

1

5
Reboot — machine boots straight into BizFlow Kiosk

Manager enrolls employees on the kiosk

Updates — zero-touch

↓

↓
Applies on next restart — zero manual steps

Roadmap

V1 ships fast. Then we expand.

The kiosk shell is designed so each new module is a self-contained addition — zero changes to auth or core shell.

V1 — Now
Biometric Attendance
Clock in/out · Fingerprint + NFC+PIN · Manager enrollment · Today's punches
V2 — Next
Inventory Spot-Counts
Employee reports current stock level for a product at the station — feeds into inventory module
V3 — Future
Employee Requests
Leave requests, shift swaps, early exit — submitted at the kiosk, approved in the main ERP by management
V4 — Future
Opening / Closing Checklists
Checklist items displayed at kiosk, employee marks completion with their biometric identity attached

Adding a module = 3 steps

1
Create a new Base44 function for the module's data
2
Create a new React component under src/kiosk/modules/
3
Register a tile in the MODULES array — it appears for the right roles
Auth shell, hardware bridge, auto-logout, and Base44 client are fully shared — never touched again.
🔌 Scale port (nirosta enclosure) reserved for future direct weight-logging from kiosk — plug in USB scale, add one module.
